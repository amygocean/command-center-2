# UI update: resizable Girls columns, cleaner calendar filters, one shoot by default

## Easiest installation

Replace the project with the contents of `new-updated.zip`, or replace these five files in the existing project:

- `index.html`
- `styles.css`
- `js/people.js`
- `js/calendar.js`
- `js/content.js`

Do not paste only the CSS from the earlier Grok answer. The resize behaviour needs matching HTML generated by `people.js`, CSS, and JavaScript.

## What changed

### 1. The Girls columns can now be resized

On a desktop-sized screen, move the pointer over the divider between two task columns. The divider turns teal. Drag it right to enlarge that column or left to make it smaller.

The selected column gains or gives up space while the other visible columns resize proportionally. The widths are saved in browser `localStorage`, so completing a task or rerendering the board does not reset them.

Double-click any divider to return all visible columns to equal widths.

On screens 960px wide or smaller, drag-resizing is disabled. Columns become swipeable cards instead, because a precise resize handle is frustrating on touch screens.

Relevant code after this update:

- `js/people.js`: lines 13–27, 110–133, and 228–318
- `styles.css`: lines 758–840
- `index.html`: line 119

### 2. Calendar project and layer filters are cleaner

Projects and optional calendar layers now sit in two labelled groups. Active items are clean white or softly tinted pills; inactive items are outlined and muted instead of crossed out.

Relevant code:

- `js/calendar.js`: lines 35–71
- `styles.css`: lines 185–212, plus the responsive rule in the main 860px media query

### 3. Studio shows one upcoming shoot by default

Studio now shows only the nearest future shoot day. `View more` reveals all other upcoming shoots and previous shoots. If there are no future shoots, it says so and still allows previous shoots to be opened.

Relevant code:

- `js/content.js`: lines 11 and 31–59

### 4. Cache-busting version was updated

`index.html` now requests version `20260718c`, which helps prevent the browser from displaying older cached CSS or JavaScript after deployment.
